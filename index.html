3. Functional Description

Expected Input:

  • Plaintext: Alphabetic characters (A-Z). No numerical or special characters.
  • Shift: Numerical characters (0-26). No alphabetic or special characters.
  • Direction: A boolean value. If the user inputs 'forward', it returns true; if 'backward', it returns false.

Processing:

  • Gather inputs (plaintext, shift, and shift direction) from the user.
  • Validate user input (If valid, proceed to the next step. If not, prompt the user to input valid data).
  • Temporarily store the inputs from the user.
  • Start the encryption process using the stored inputs.
  • Display the encrypted message.

Output:

The encrypted message is displayed on the same webpage (localhost) below the input text boxes, along with the unencrypted message.

4. Security Considerations

Vulnerability Assessment

In any web application, it’s crucial to identify potential security risks. For the Caesar Cipher program, the following vulnerabilities may be present:

  • Input Validation Vulnerabilities: The application accepts user input for plaintext, shift value, and direction. If not properly validated, this may lead to unexpected behavior or security risks, such as code injection or denial of service.
  • Cross-Site Scripting (XSS): If user input is rendered without adequate sanitization, malicious users may inject scripts that could execute in the context of another user’s browser session.
  • Server Misconfiguration: Running the Flask development server in production exposes the application to various risks, such as exposing sensitive information or providing unnecessary access to system resources.

Mitigation Strategies

To address the identified vulnerabilities, the following measures should be implemented:

  • Input Validation: Ensure that all user inputs are validated thoroughly. For instance:
    • Plaintext must only consist of alphabetic characters (A-Z).
    • Shift values must be restricted to numerical characters between 0 and 26.
    • Implement error handling to manage invalid inputs effectively and prompt users for correction.
  • Sanitization: Use Flask's built-in tools to escape user inputs before rendering them in the HTML template. This helps prevent XSS attacks.
  • Use of a Production Server: When deploying the application, utilize a production-ready WSGI server (such as Gunicorn or uWSGI) instead of the built-in Flask server to minimize risks associated with server misconfiguration.
  • Environment Variables: Store sensitive configurations (e.g., secret keys) in environment variables rather than hardcoding them into the application.

Testing

To verify the security measures implemented, the following testing procedures should be conducted:

  • Unit Testing: Write unit tests for input validation functions to ensure they correctly handle valid and invalid inputs.
  • Penetration Testing: Conduct penetration testing to simulate potential attacks (e.g., XSS, SQL injection) and identify weaknesses in the application’s security.
  • Static Code Analysis: Utilize tools like Bandit or PyLint to analyze the codebase for common security vulnerabilities and best practices.
  • Code Reviews: Perform peer code reviews focusing on security aspects to catch potential vulnerabilities before deployment.

5. Usage Instructions

  • Install Python from the official site: https://python.org
  • Install Flask: 
    pip install flask
  • (Optional) Set up a virtual environment: 
    python -m venv venv
source venv/bin/activate  # On macOS/Linux
venv\Scripts\activate  # On Windows
  • Navigate to the script directory.
  • Activate the virtual environment (if created).
  • Run the Flask application: 
    python app.py
  • Open a web browser and go to http://127.0.0.1:5000.
  • Enter the plaintext, shift value, and select the direction.
  • Click on the submit button to see the result.

6. Error Handling

User Input Validation:

If the user inputs unexpected or unwanted characters, submission of the inputs will be denied, and the system will prompt the user for valid inputs.

Flask:

If Flask is not installed on the device, errors may occur. To resolve this, open Command Prompt as an administrator and type:

pip install flask

Running in CMD:

To avoid errors, make sure CMD is run as administrator. If issues persist, try opening the file in a virtual environment. To create a virtual environment, use the following command:

python3 -m venv venv

After that, activate the virtual environment:

source venv/bin/activate
